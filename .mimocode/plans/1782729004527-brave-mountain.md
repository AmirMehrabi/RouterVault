# SkyBase SaaS MVP Implementation Plan

## Overview

Convert the existing MikroTik controller into a subscription-based SaaS product. The current multi-tenant Stancl architecture is preserved. A SaaS subscription layer is added on top with plan enforcement, router limits, overage billing, onboarding wizard, HTTPS push backups, and tiered alert features.

---

## Architecture Decisions

- **Tenancy**: Keep Stancl per-tenant DB multi-tenancy as-is
- **Extra routers**: Allow overage billing at $1/router/month beyond plan limits
- **Onboarding**: Setup wizard after registration (pick plan → add first router → configure backup)
- **Backup methods**: Both SSH pull (existing) and HTTPS push (new script endpoint)
- **Alerts**: All plans get alerts; tiered by channel (in-app → email → Telegram/webhook)
- **Payment gateway**: Dummy payment page for MVP (no real gateway)

## Phase 4: HTTPS Push Backup (Router Script)

### 4.1 Create BackupUploadController

**File**: `app/Http/Controllers/BackupUploadController.php`

Public API endpoint (no auth, token-based) for routers to push config:

```php
// routes/api.php
Route::post('/v1/backups/upload', [BackupUploadController::class, 'upload'])
    ->middleware('throttle:10,1');
```

The controller:
1. Validates the upload token (stored in `backup_tokens` table)
2. Accepts `.rsc` file upload
3. Creates RouterBackup record
4. Runs diff against previous backup
5. Generates alerts if changes detected
6. Returns JSON response

### 4.2 New Migration: Create backup_tokens table

**File**: `database/migrations/2026_06_29_000005_create_backup_tokens_table.php`

```php
Schema::create('backup_tokens', function (Blueprint $table) {
    $table->id();
    $table->string('tenant_id');
    $table->foreignId('router_id');
    $table->string('token', 64)->unique();
    $table->boolean('is_active')->default(true);
    $table->datetime('last_used_at')->nullable();
    $table->timestamps();
});
```

### 4.3 Generate RouterOS Push Script

**File**: `app/Services/Backups/RouterPushScriptGenerator.php`

Generates the RouterOS script that:
1. Runs `/export hide-sensitive`
2. Uploads the `.rsc` file to the SaaS HTTPS endpoint
3. Includes the backup token for authentication

Script template:
```
/tool fetch url="https://cloud.skybase.app/api/v1/backups/upload" \\
    http-method=post \\
    http-data=":rsc file content" \\
    http-header-field="Authorization: Bearer TOKEN"
```

### 5.3 Create AlertNotificationService

**File**: `app/Services/Saas/AlertNotificationService.php`

```php
class AlertNotificationService
{
    public function send(DiffAlert $alert, Tenant $tenant): void
    {
        $channels = $tenant->alert_channels;
        
        // Always create in-app notification
        $this->createInAppNotification($alert);
        
        // Email if plan includes it
        if (in_array('email', $channels)) {
            $this->sendEmail($alert, $tenant);
        }
        
        // Telegram if plan includes it
        if (in_array('telegram', $channels)) {
            $this->sendTelegram($alert, $tenant);
        }
    }
}
```

## Phase 8: Security & Branding

### 8.1 Backup Token Security

- Tokens are 64-char random strings, never exposed in UI
- Tokens can be regenerated per-router
- Rate limiting on upload endpoint (10 req/min)
- Validate file type (.rsc only)
